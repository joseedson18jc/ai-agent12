# Python Patterns
Pythonic idioms, type hints, async/await, virtual environments, PEP compliance.
## Instructions
- Use type hints (PEP 484) for all function signatures
- Follow PEP 8 for style, PEP 257 for docstrings
- Use async/await for I/O-bound operations
- Manage dependencies with venv + requirements.txt or Poetry
- Prefer list comprehensions over map/filter
- Use dataclasses or Pydantic for structured data
