# Systematic Debugging
Four-phase root cause process: reproduce, isolate, identify, verify fix.
## Instructions
1. REPRODUCE: Create a minimal reproduction case
2. ISOLATE: Narrow down to the specific component/line
3. IDENTIFY: Determine the root cause (not just symptoms)
4. VERIFY: Confirm the fix resolves the issue and doesn't regress
- Document findings for future reference
- Add regression tests for the fix
